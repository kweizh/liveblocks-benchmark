# Liveblocks Emoji Reaction Broadcast ## Background Build a Next.js page that lets multiple users in the same Liveblocks room send floating emoji reactions to each other. This evaluates your ability to use Liveblocks v2 broadcast events (`useBroadcastEvent` and `useEventListener`) to deliver ephemeral, non-persisted realtime signals between clients. ## Requirements * Implement a single-page Next.js app at `/` that joins a fixed Liveblocks room. * Render four clickable emoji buttons: 👍, ❤️, 🎉, 😂. * Clicking a button must broadcast a custom event of shape `{ type: "REACTION", emoji: "<emoji>" }` to all other users in the room. * Receiving clients must visibly display the received emoji somewhere on the page as a floating reaction element. The element must remain in the DOM for at least 2 seconds and then disappear (fade or be removed) after roughly 2 seconds. * Each floating reaction element must carry the CSS class `reaction` and the attribute `data-testid="floating-reaction"`, and its text content must include the emoji character. * Use real Liveblocks credentials read from environment variables. No mocking of Liveblocks. ## Implementation Hints * Use Liveblocks v2 React bindings: import from `@liveblocks/react` and `@liveblocks/client`. * Wrap the page in a `LiveblocksProvider` (configured with the public API key) and a `RoomProvider`. The public key must come from the env var with the `NEXT_PUBLIC_` prefix so it is available to client components. * Use `useBroadcastEvent()` to send events from the clicking client. * Use `useEventListener()` to receive events on other clients.
