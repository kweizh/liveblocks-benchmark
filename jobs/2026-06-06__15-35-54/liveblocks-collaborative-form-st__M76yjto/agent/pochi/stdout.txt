The project is a Next.js (App Router) application using the Liveblocks v2 React SDK (`@liveblocks/client`, `@liveblocks/react`). Once submitted, the page should clearly show a submitted state that is visible to every connected user (e.g. a visible element with the text `Submitted`). * The Liveblocks room id must be `harbor-form-${NEXT_PUBLIC_ZEALT_RUN_ID}` so concurrent runs do not collide. * All Liveblocks connectivity must use the real Liveblocks service via the public key — do not mock or stub the client. ## Implementation Hints * Install `@liveblocks/client` and `@liveblocks/react` (v2.x) alongside Next.js. * Use `createClient({ publicApiKey: process.env.NEXT_PUBLIC_LIVEBLOCKS_PUBLIC_KEY! })` in a client component and wrap the page with `RoomProvider` from `@liveblocks/react`, passing `initialStorage` so the `form` LiveObject is seeded with empty strings and `submitted: false` on first connect. * Read individual form fields with `useStorage((root) => root.form.name)` (and similar), and write them via `useMutation(({ storage }, key, value) => { storage.get('form').set(key, value); }, [])`. * Re-export the run id at dev-server start time, e.g. `NEXT_PUBLIC_ZEALT_RUN_ID=$ZEALT_RUN_ID npm run dev`. * Make sure each input has a stable, queryable identifier such as `data-testid="field-name"`, `data-testid="field-email"`, `data-testid="field-message"`, the submit button has `data-testid="submit"`, and the submitted indicator has `data-testid="submitted"`. * After verifying things work locally, KILL the dev server before finishing — the verifier will start its own server.
