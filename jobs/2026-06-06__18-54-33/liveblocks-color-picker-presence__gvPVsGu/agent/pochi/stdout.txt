## How It Works

1. **Auth**: Every client POSTs to `/api/liveblocks-auth` → server signs an access token with `LIVEBLOCKS_SECRET_KEY` and grants room access.
2. **Presence**: Each user's `presence.color` starts at `"#000000"`. Changing the color input updates it via `useUpdateMyPresence`.
3. **Swatches**: `useSelf` and `useOthers` are combined into `allUsers`; each entry renders an `<li>` with `data-connection-id`, `data-color`, and `background-color` matching the hex value.
4. **Room ID**: Derived at runtime as `` `color-picker-${process.env.NEXT_PUBLIC_ZEALT_RUN_ID}` ``.
