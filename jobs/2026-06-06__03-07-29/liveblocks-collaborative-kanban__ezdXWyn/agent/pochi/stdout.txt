# Liveblocks Collaborative Kanban Board (Next.js) ## Background This Next.js project contains a scaffold for a real-time collaborative Kanban board powered by https://liveblocks.io/. The board has three fixed columns (`To Do`, `In Progress`, `Done`) and stores its data inside a shared Liveblocks room so that any change made by one user is instantly synchronized to every other user in the same room. The scaffold already renders the empty columns and wires up the `RoomProvider`, but it does not yet implement card creation, editing, deletion, or drag-and-drop movement between columns. Your job is to finish the Kanban application using **Liveblocks Storage** (`LiveList`, `LiveObject`, `LiveMap`) and the `useMutation` hook. The Liveblocks **cloud service is used directly** — there is no mocking. The provided keys belong to a real Liveblocks project. ## Requirements Implement the following features inside the scaffold at `/home/user/myproject`: 1. **Shared storage schema** (do not change the keys/shape): * `Storage.columns: LiveList<LiveObject<{ id: string; title: string; cardIds: LiveList }>>` initialized with exactly three columns whose `id` values are `todo`, `in-progress`, `done` and whose `title` values are `To Do`, `In Progress`, `Done` (in that order). * `Storage.cards: LiveMap<string, LiveObject<{ id: string; title: string; description: string; assignee: string }>>`. 2. **Add Card**: Each column has an input + button (or equivalent) that adds a new card to that column. The new card's `id` must be appended to that column's `cardIds` and the card itself must be inserted into `Storage.cards`. 3. **Inline edit card title**: Clicking the card title turns it into an editable input. Saving (Enter or blur) writes the new title back to the corresponding `LiveObject` in `Storage.cards`. 4. **Delete card**: Each card has a delete control that removes the card's id from its column's `cardIds` and deletes the entry from `Storage.cards`. 5. **Drag-and-drop between columns**: Cards can be moved between any two columns using drag-and-drop (HTML5 DnD or `react-dnd`, your choice). The move **MUST** happen inside a single `useMutation` call that (a) removes the card id from the source column's `cardIds` and (b) pushes/inserts it into the target column's `cardIds`, so the operation is batched as a single Liveblocks transaction. 6. **Real-time sync via Liveblocks cloud**: All changes must be performed through Liveblocks Storage (no local-only state). Two browser tabs joined to the same room must see each other's changes in real time. 7. **Auth endpoint**: Implement (or keep) a server route at `POST /api/liveblocks-auth` that calls `Liveblocks.prepareSession` (or `identifyUser`) using the `LIVEBLOCKS_SECRET_KEY` environment variable and grants full access to rooms whose id starts with `kanban-`. 8. **Room id**: The board page must join the Liveblocks room `kanban-${ZEALT_RUN_ID}`. The harness only injects `ZEALT_RUN_ID` (server-side) into the container; the browser bundle cannot read it directly. You **MUST** mirror it into the client by re-exporting it as `NEXT_PUBLIC_ZEALT_RUN_ID` (e.g. set `env.NEXT_PUBLIC_ZEALT_RUN_ID = process.env.ZEALT_RUN_ID` inside `next.config.js`, or write it into `.env`/`.env.local` during build, so that `process.env.NEXT_PUBLIC_ZEALT_RUN_ID === process.env.ZEALT_RUN_ID` at runtime). Fall back to the literal string `kanban-local` only when neither variable is set.
