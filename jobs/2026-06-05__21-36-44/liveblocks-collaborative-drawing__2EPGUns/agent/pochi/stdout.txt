### 3. `components/CanvasScaffold.tsx` (fully rewritten)
The placeholder scaffold is replaced with a complete implementation:

- **`useStorage`** — subscribes to `root.strokes` (a `LiveList<LiveObject<Stroke>>`) and re-renders whenever any stroke or point changes.
- **`useMutation` — `startStroke`** — called on `pointerdown`; creates a new `LiveObject` with the current `userId`, `color`, `width`, a `nanoid()` id, and an initial `LiveList<{x,y}>` point, then pushes it to the shared `strokes` list.
- **`useMutation` — `addPoint`** — called on `pointermove` with a **~30 ms timestamp throttle** (at most one Liveblocks write per 30 ms); appends a `LiveObject({x,y})` to the active stroke's `points` list.
- **`useMutation` — `deleteStroke`** — used by the eraser; finds a stroke by `id` and calls `LiveList.delete(index)`.
- **`useMutation` — `undoMine`** — walks the `LiveList` backwards, finds the last stroke where `userId` matches the current user, and deletes it. Other users' strokes are never touched.
- **SVG rendering** — maps each stroke to `<polyline data-stroke-id={id} data-user-id={userId} points="x1,y1 x2,y2 ..." stroke={color} strokeWidth={width} />`. The polyline uses `pointerEvents: "stroke"` when the eraser is active so click-to-erase targets the visible stroke path only.
- **Pointer capture** — `setPointerCapture` on `pointerdown` so `pointermove` events are received even if the cursor leaves the SVG during a fast stroke.
- **Coordinate mapping** — pointer client coordinates are scaled from the element's bounding rect to the SVG viewBox (1024×720), so drawing is accurate regardless of how the SVG is scaled by CSS.

All required `data-testid` attributes are present: `canvas`, `color-picker`, `width-picker`, `tool-pen`, `tool-eraser`, `undo-mine`. The server runs on port 3000 with zero TypeScript errors.
